System Requirements

Minimum Requirements

  • 2GB RAM (4GB+ recommended)
  • 500MB available disk space
  • Intel Core i5 or equivalent (2013+)
  • Modern web browser (Chrome 90+, Firefox 88+, Safari 14+)

Platform-Specific

Platform Version Processor Notes
macOS 10.15 Catalina+ Intel or Apple Silicon Universal binary included
Linux Ubuntu 20.04+, Fedora 32+ x64 glibc 2.31+
Windows Windows 10 / 11 x64 .NET Framework 4.5+

Network Requirements

  • Internet connection to reach GPU server (142.54.161.210:11434)
  • Firewall rule allowing outbound on port 11434
  • Optional: VPN if using corporate network

Check Your System

macOS: Click Apple menu → About This Mac
Linux: uname -a
Windows: Settings → System → About

Installation

macOS

  1. Download the appropriate installer (Intel or Apple Silicon)
  2. Open the downloaded .dmg file
  3. Drag "Ollama VS Code" to the Applications folder
  4. Open Applications and launch "Ollama VS Code"
  5. Grant microphone permission when prompted (for voice commands)
✓ Installation Complete - VS Code Ollama should open automatically.

Linux

# Extract the archive
unzip ollama-vscode-linux-x64.zip -d ~/.local/share/

# Create desktop shortcut (optional)
cp ollama-vscode.desktop ~/.local/share/applications/

# Launch
~/.local/share/ollama-vscode/bin/code
💡 Tip: Add to PATH for easy command-line access: export PATH=$PATH:~/.local/share/ollama-vscode/bin

Windows

  1. Download the Windows Installer
  2. Run ollama-vscode-setup.exe
  3. Follow the installation wizard
  4. Grant microphone permission when prompted
  5. Launch from Start menu or desktop shortcut
⚠️ Windows Defender: May show a warning. Click "More info" then "Run anyway" to proceed.

GPU Server Configuration

Option 1: Use Public GPU Server (Default)

The default configuration connects to Rocha Studios' public GPU server at 142.54.161.210:11434. This is qwen3:14b, optimized for fast completions.

Verify Connection

Open VS Code and run:
Ctrl + Shift + P (Windows/Linux) or Cmd + Shift + P (macOS)
Type: Ollama: Check Status

Option 2: Run Your Own GPU Server

For privacy or performance, run Ollama locally on your own GPU hardware.

Requirements

  • NVIDIA GPU with CUDA 11.8+ OR AMD with ROCm OR macOS with Metal (M1+)
  • 8GB+ VRAM recommended (14GB for qwen3:14b)
  • 30GB disk space for model

Installation

# macOS / Linux
curl -fsSL https://ollama.ai/install.sh | sh

# Windows (download from https://ollama.ai/download)
# Then run: ollama serve

# Pull the model
ollama pull qwen3:14b

# Start server
ollama serve

Configure VS Code to Use Local Server

  1. Open VS Code Settings (Cmd/Ctrl + ,)
  2. Search: "Ollama Host"
  3. Set to: http://localhost:11434
  4. Run "Ollama: Check Status" to verify
✓ GPU Server Ready - Completions will now use your local server.

Configuration

Essential Settings

1. Host & Port

{
  "vscodeOllama.host": "http://142.54.161.210",  // Or localhost:11434 for local
  "vscodeOllama.port": 11434
}

2. Model Selection

{
  "vscodeOllama.model": "qwen3:14b"  // Available: mistral:7b, neural-chat, llama3.1:70b
}

3. Completion Quality

{
  "vscodeOllama.temperature": 0.3,      // 0.0 = deterministic, 1.0 = creative
  "vscodeOllama.maxTokens": 100,        // Longer = more detailed completions
  "vscodeOllama.debounceMs": 300,       // Wait 300ms after typing to request
  "vscodeOllama.timeout": 5000          // Request timeout in milliseconds
}

4. Features

{
  "vscodeOllama.qaEnabled": true,           // Auto security analysis
  "vscodeOllama.showDailySuggestions": true // Daily tips
}

Performance Tuning

Scenario Recommendation
Slow Network Increase debounceMs to 500-800ms, reduce maxTokens to 50
Fast Network Decrease debounceMs to 100-200ms, increase maxTokens to 150
High Latency Increase timeout to 10000-15000ms
Less CPU Usage Set temperature to 0.1, increase debounceMs

Chrome Extension Setup

Installation

  1. Open Chrome and go to chrome://extensions
  2. Enable "Developer mode" (top right)
  3. Click "Load unpacked"
  4. Navigate to the extension folder in your VS Code installation
  5. Click "Select" to load the extension
💡 Tip: Production install via Chrome Web Store coming soon. For now, development mode is required.

Grant Microphone Permission

  1. Click the Ollama Voice Assistant icon in your toolbar
  2. Click "Start Listening"
  3. Browser will prompt for microphone access
  4. Click "Allow" to proceed

Configure Language

Default is English (US). Change in extension settings (coming in v1.1).

Verify Installation

Checklist

✓ All systems ready when:

  • VS Code launches without errors
  • "Ollama: Check Status" shows green checkmark
  • Typing triggers completion suggestions (inline)
  • Chrome extension appears in toolbar
  • Saying "code test" inserts text in editor

Test Completion

  1. Create a new file: test.js
  2. Type: function hello and wait 1-2 seconds
  3. Inline suggestion should appear
  4. Press Tab to accept

Test Voice

  2. Click "Start Listening"
  3. Say: "code console.log"
  4. Text should appear in editor

Troubleshooting

No completions appearing

Solution: Run "Ollama: Check Status". If it fails, check host/port settings. Verify network can reach 142.54.161.210:11434. Try a simple test file to rule out language-specific issues.

Slow or hanging completions

Solution: Check network latency: ping 142.54.161.210. Increase timeout in settings if latency >200ms. Reduce maxTokens to 50. Check if GPU server is overloaded (try again in 1 minute).

Voice not working

Solution: Check microphone in system settings. Grant permission in browser (chrome://settings/content/microphone). Verify localhost:8080 is allowed. Try different browser if Chrome fails.

Extension not loading

Solution: Verify Developer Mode is enabled. Check console (F12) for errors. Ensure all files are present in extension folder. Try removing and re-adding.

VS Code crashes on startup

Solution: Delete settings file: ~/.config/Code/settings.json (Linux). Restart. Disable extensions except Ollama. Check VS Code version (need 1.85+).

Port already in use

Solution: If running local Ollama and 11434 is taken, use different port. Edit config: vscodeOllama.port: 11435. Then run: ollama serve --port 11435

Microphone permission denied

Solution: macOS: System Preferences → Security & Privacy → Microphone. Windows: Settings → Privacy → Microphone. Linux: PulseAudio settings. Clear browser site data and retry.

QA analyzer not detecting issues

Solution: QA only works on PHP files (especially WPBasic). Check file extension. Verify qaEnabled is true in settings. PHP issues must match security patterns (escaping, nonces, SQL injection).
